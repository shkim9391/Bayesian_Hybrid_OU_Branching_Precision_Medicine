# Bayesian stochastic modeling of stage-specific evolutionary dynamics in pediatric leukemia

This repository contains the complete computational workflow for the manuscript:

> **Stage-specific evolutionary dynamics in pediatric leukemia inferred using Bayesian stochastic modeling**

The study applies Ornstein–Uhlenbeck (OU) stochastic-process modeling to sparse and irregular longitudinal measurements from pediatric KMT2A-rearranged leukemia. The repository includes data preprocessing, simulation-based calibration, Bayesian posterior inference, posterior predictive evaluation, model diagnostics, and generation of all principal and supplementary figures and tables.

## Overview

Longitudinal cancer measurements are often collected at irregular intervals and contain relatively few observations per patient. We therefore used an exact OU transition-density formulation that accounts directly for the elapsed time between consecutive observations.

For patient \(i\), the modeled process was

\[
dX_i(t)
=
\theta_{g[i]}
\left[
\mu_i-X_i(t)
\right]dt
+
\sigma_{g[i]}dW_i(t),
\]

where:

- \(\mu_i\) is the patient-specific equilibrium or attractor level;
- \(\theta_{g[i]}\) is the group-level mean-reversion rate;
- \(\sigma_{g[i]}\) is the group-level diffusion scale;
- \(g[i]\) denotes the clinical-stage or outcome group;
- \(W_i(t)\) is a standard Wiener process.

For observations separated by \(\Delta t\), the exact transition distribution was

\[
X(t+\Delta t)\mid X(t)
\sim
\mathcal{N}
\left(
\mu + [X(t)-\mu]e^{-\theta\Delta t},
\frac{\sigma^2}{2\theta}
[1-e^{-2\theta\Delta t}]
\right).
\]

An additional residual-error term was included in the Bayesian observation model.

## Repository organization

- `data/raw/` contains the original longitudinal input data, when redistribution is permitted.
- `data/processed/` contains cleaned patient trajectories and analysis-ready metadata.
- `data/supplementary_data/` contains the numbered supporting datasets accompanying the article.
- `scripts/` contains the executable analysis scripts in workflow order.
- `outputs/` contains the direct outputs generated by each script.
- `manuscript/` contains publication-formatted figures and supplementary tables.
- `docs/` contains data dictionaries, workflow documentation, and output mappings.
- `tests/` contains basic validation tests for the transition density, input files, and expected outputs.

## Input data

The primary input file is:

The preprocessing workflow produces:

data/processed/kmt2a_longitudinal_clean.xlsx
data/processed/series_auto.csv
data/processed/series_auto_debug.csv
data/processed/longitudinal_metadata_debug.csv
data/processed/patient_group.csv

The primary analysis time variable is expressed in years:

[
t_{\mathrm{years}} = \frac{\mathrm{Day}}{365.25}.
]

Measurements recorded for the same patient on the same day are collapsed by taking their arithmetic mean. No temporal interpolation or imputation is used for the hierarchical Bayesian analysis.

See docs/data_dictionary.md for variable definitions.

## Posterior predictive evaluation

Posterior predictive trajectories were generated from the fitted posterior distributions and compared with the observed longitudinal measurements.

Model performance was summarized using:

* root mean squared error;
* the proportion of observations within the patient-level 95% posterior predictive interval;
* patient-level and group-level posterior predictive summaries;
* relationships between sampling characteristics and model diagnostic measures.

## Reproducibility

Random seeds are specified within the simulation and posterior-sampling scripts. Software versions are recorded in environment.yml, and model-run settings are exported to:

ou_direct_transition_run_metadata.csv

The posterior trace is stored in NetCDF format so that figures and tables can be regenerated without repeating the full MCMC fit.

Small numerical differences may occur across operating systems or software-library builds because of differences in random-number generation, floating-point arithmetic, and parallel MCMC execution.
